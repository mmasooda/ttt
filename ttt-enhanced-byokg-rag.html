🚀 TTT-Enhanced BYOKG-RAG System

📐 Architecture Overview

This enhanced system integrates Test-Time Training (TTT) with your existing BYOKG-RAG framework to improve BOQ generation accuracy through adaptive learning at inference time.

Key Enhancements:

  • Test-Time Adaptation: Fine-tunes the model on query-specific examples during inference
  • Leave-One-Out Training: Creates synthetic training tasks from your provided examples
  • LoRA Adapters: Efficient parameter updates without modifying the base model
  • Multi-Strategy Retrieval: Enhanced graph traversal with TTT-informed ranking
  • Iterative Refinement: Improved orchestration with TTT feedback loops

🔄 TTT Integration Flow

  1. Query Analysis: Enhanced KG-Linker with TTT-adapted entity recognition
  2. Example Collection: System requests 10 relevant examples from user
  3. TTT Training: Creates leave-one-out tasks and fine-tunes adapters
  4. Enhanced Retrieval: Uses adapted model for better graph queries
  5. BOQ Generation: Produces more accurate bills with TTT-informed decisions

⚠️ Implementation Notes

  • The system will prompt you for 10 example BOQs during first run
  • AWS S3 credentials and API keys will be requested during setup
  • LoRA adapters are cached for efficiency (configurable)
  • Requires GPU for optimal TTT performance (CPU fallback available)

🎯 Key Features

  • Adaptive Learning: Model improves with each query through TTT
  • Microservices Architecture: Clean separation of concerns
  • Scalable Design: Supports distributed training and inference
  • Production Ready: Docker support with health checks and monitoring
  • Extensible Framework: Easy to add new adapters and strategies
